## Code of Conduct

By participating in this project, you agree to:

- **Be respectful and constructive** — Assume good faith. Every contributor is trying to help.
- **Focus on the content, not the person** — Critique the book, not the author.
- **Use inclusive language** — Ontology engineering is for everyone.
- **Follow the principle of charitable interpretation** — If a statement could be read in multiple ways, assume the author meant the most reasonable one.

Rude, dismissive, or personally attacking comments will not be tolerated.

## What Kind of Contributions Are Welcome

Almost everything is welcome, including:

| Category | Examples |
| :--- | :--- |
| **Typographical fixes** | Spelling errors, missing words, incorrect punctuation |
| **Grammatical improvements** | Awkward phrasing, unclear sentences |
| **Technical corrections** | Incorrect OWL syntax, misleading logical statements, reasoning behavior errors |
| **Conceptual clarifications** | Better analogies, additional examples, clearer distinctions |
| **Structural improvements** | Broken cross-references, missing table of contents entries |
| **Outdated patterns** | Deprecated Protégé features, changed reasoner behavior, OWL 2 updates |
| **Pedagogical suggestions** | "This section took me three reads — here's why" |
| **Translation contributions** | Help translate chapters to other languages (future) |

**Not currently welcome:**

- Spam or self-promotion
- Entirely new chapters without prior discussion (open an issue first)
- Changes that break existing correct semantics

## How to Contribute

You have two primary ways to contribute, depending on your comfort level.

### Quick Comparison

| Method | Best for | GitHub experience needed |
| :--- | :--- | :--- |
| **GitHub Issue** | Reporting errors, suggesting improvements, asking questions | Minimal (just a GitHub account) |
| **Pull Request** | Directly fixing typos, code examples, or small clarifications | Moderate (fork, edit, commit, push) |

Both methods are appreciated equally. One is not "better" than the other.

### If You Are New to GitHub

1. Create a free GitHub account at https://github.com
2. Navigate to the repository: https://github.com/yasenstar/protege_pizza
3. Click the **Issues** tab
4. Click **New Issue**
5. Use the provided template
6. Submit

That's it. No Git commands needed.

## Reporting Errors (GitHub Issues)

### Step 1: Check if the error already exists

Search **open and closed issues** before creating a new one. Duplicate reports waste time.

### Step 2: Use the template

The repository has a pre-configured issue template. Please fill it out completely.

**Template:**

```markdown
## Error Report

**Chapter:** [e.g., Chapter 14]
**Section:** [e.g., 14.5.2]
**Page/Line:** [if applicable]

**Error Type:** (Technical / Conceptual / Typographical / Outdated / Pedagogical)

**Current Text:**
> [Quote the exact text that contains the error]

**Suggested Correction:**
> [Describe what should replace it, or why it is wrong]

**Reasoning:**
> [Explain why this is an error — cite OWL specification, reasoner behavior, or logical principles]

**Additional Context:**
> [Protégé version, reasoner used, screenshots if helpful]
